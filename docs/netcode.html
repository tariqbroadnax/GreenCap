Netcode

The network consists of two actors, the client and server.

Client

On initialization, the client attempts to connect to the server. If the client cannot connect, then an IOException is thrown and nothing is loaded. Otherwise, the clients recieves their player's name and loads everything else.

Every frame, the client checks whether they have recieved and/or should send commands.

Once the recieves a command, it updates the local scene.

Whenever the user presses an arrow key, a move command is created. First it updates the local scene. This is a form interpolation that makes the program feel tight. After, the client sends the commands to server. The command is sent as a string in the following format:

move [movex] [movey]

The movex and movey specify the direction that the player is moving.

Server

The Server runs on two threads: the connector thread and the update thread.

The connector thread connects with clients. It blocks until a client attempts to connect with it. After it creates a unique player name and sends it to the client. It then continues to block and listen.

The update thread obviously updates. It first checks if a client has sent a command and executes them. It then updates the server's scene. After it sends each client a update command for each player. The command is in the following format.

player [playerAttr0] [playerAttr1] ...

In the event that any of this fails, the server disconnects with client immediately.
